# Artifact Contract

Write these artifacts under the run output directory (`--out-dir`).

## Run directory layout

```
<out-dir>/
├── run-summary.json              # orchestrate-e2e JSON artifact (base schema + gates array)
├── review-target.json            # gate 1: target readiness
├── plan-experiment.json          # gate 2: experiment plan
├── check-dataset-quality.json    # gate 3: dataset audit
├── check-data-pipeline.json      # gate 4: pipeline validation
├── feature-engineer.json         # gate 4b: feature contract (when invoked)
├── babysit-training.json         # gate 5: training monitoring
├── check-failed-run.json         # gate 5b: failure diagnosis (when invoked)
├── check-eval.json               # gate 6: evaluation results
├── explain-model.json            # gate 7: explainability and promotion
├── report.html                   # generated viewer (from generate_report.py)
├── run-contract.md               # existing: objective, label, metric, threshold, budget
├── gate-status.md                # existing: gate status table with PASS/FAIL/SKIPPED
├── run-timeline.md               # existing: chronological action log
├── evaluation-summary.md         # existing: metric table and baseline deltas
└── decision.md                   # existing: final GO/NO-GO with blockers and next commands
```

## Required markdown files

1. `run-contract.md`
   - Objective, label, metric, threshold, budget, retry policy, promotion criteria

2. `gate-status.md`
   - One row per lifecycle gate with status (`PASS`, `FAIL`, `SKIPPED`)
   - Evidence path(s) and timestamp

3. `run-timeline.md`
   - Chronological log of key actions, failures, retries, and decisions

4. `evaluation-summary.md`
   - Metric table, baseline deltas, regressions, and confidence notes

5. `decision.md`
   - Final `GO` or `NO-GO`
   - Blockers/risks and exact next commands

## Required JSON artifacts

Each sub-skill writes its own JSON artifact to `--out-dir`. See [../../../../references/schemas.md](../../../../references/schemas.md) for the full schema definitions and [../../../../references/vocabulary.md](../../../../references/vocabulary.md) for canonical enum values.

The `run_id` field in every artifact must match the one generated by `orchestrate-e2e` at run start.

## Generated artifacts

`report.html` is generated by `generate_report.py` after all gates complete:

```bash
uv run plugins/agentic-ml/report-viewer/generate_report.py <out-dir>
```

## Quality bar

- Use absolute file paths for artifact references.
- Include at least one reproducible command per failed gate.
- Keep decision rationale concise and evidence-backed.
- All JSON artifacts must include `schema_version: "1.0"` and `run_id`.
